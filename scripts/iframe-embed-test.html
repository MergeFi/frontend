MergeFi clickjacking / anti-embedding check

This page plays the role of a third-party site trying to iframe MergeFi. With X-Frame-Options: DENY in place, the browser must refuse to render the frame below — check the devtools console for a "Refused to display '...' in a frame because it set 'X-Frame-Options' to 'deny'" message, and the frame area should stay blank instead of showing the app.

Usage: npm run build && npm start in one terminal, then open this file directly in a browser (file:// is fine — X-Frame-Options applies regardless of the parent's origin) and edit the src below to match the port npm start printed.
